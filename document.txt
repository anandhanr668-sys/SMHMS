## ğŸ¯ PROJECT VISION

**Problem Statement:**
- Every hospital has unique workflows, departments, and protocols
- Existing HMS software is rigid and vendor-dependent
- Changes require weeks/months and expensive vendor intervention
- No flexibility for hospitals to adapt software to their needs

**Our Solution:**
"WordPress for Hospital Software" - A reusable LCNC platform where hospitals can configure their entire system through an admin interface without coding.

## ğŸ‘¥ USER ROLES & ACCESS CONTROL

### 1. ADMIN (Full System Control)
**Primary Responsibilities:**
- Configure entire system using LCNC interface
- Design forms, protocols, reports via drag-and-drop
- Manage all master data and business rules
- View complete system analytics
- Access all patient records and history

**Key Features:**
- No-code Form Builder
- Drag-drop Report Template Designer
- Protocol & Rules Configuration
- Staff & Resource Management
- Audit Trail Viewer
- System Analytics Dashboard

---

### 2. FRONT DESK (Patient Entry Point)
**Primary Responsibilities:**
- Patient registration and check-in
- Appointment scheduling
- Basic demographic data entry
- Generate Patient ID

**Access Restrictions:**
- Cannot access medical data
- Cannot modify system configurations
- Cannot view other staff records

---

### 3. NURSE (Clinical Data Capture)
**Primary Responsibilities:**
- Capture patient vitals
- Record initial assessment
- Submit investigation requests
- View bed allocation status
- Monitor assigned patients

**Access Restrictions:**
- Cannot modify rules/protocols
- Cannot discharge patients
- Cannot access admin configurations
- View only assigned patients

---

### 4. DOCTOR (Diagnosis & Treatment)
**Primary Responsibilities:**
- View patient history and vitals
- Conduct consultation
- Prescribe medications
- Order investigations
- Discharge patients
- View past medical records

**Access Restrictions:**
- View only assigned/eligible patients
- Cannot modify forms or rules
- Cannot access admin panel
- Cannot view other doctors' private notes

---

### 5. PATIENT (Self-Service Portal)
**Primary Responsibilities:**
- View own medical records
- Download reports
- Track appointments
- View prescriptions
- Access visit history

**Access Restrictions:**
- View only own data
- Cannot modify any records
- No access to other patients
- Read-only access
